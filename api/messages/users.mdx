---
title: "Get users available for messaging"
description: "Retrieve a list of all registered users except yourself. Use this to populate the chat sidebar with contacts you can start or continue a conversation with."
---

Use this endpoint to fetch every registered user other than yourself. The response excludes the authenticated user and strips the `password` field from every returned object. This is the primary data source for the chat sidebar that lets you browse contacts and open conversations.

## Endpoint

```
GET /api/message/users
```

## Authentication

This endpoint requires a valid JWT. Pass the token in the `Authorization` header as a Bearer token.

```
Authorization: Bearer <token>
```

<Warning>
  Requests without a valid Bearer token in the `Authorization` header are rejected with `401 Unauthorized`.
</Warning>

## Request body

This endpoint does not accept a request body or query parameters.

## Response

Returns an array of user objects. Each object represents a user you can message. The `password` field is never included in any returned object.

<ResponseField name="[]._id" type="string" required>
  Unique identifier for the user document.
</ResponseField>

<ResponseField name="[].fullName" type="string" required>
  The user's full display name.
</ResponseField>

<ResponseField name="[].email" type="string" required>
  The user's email address.
</ResponseField>

<ResponseField name="[].profilePic" type="string" required>
  URL of the user's profile picture. Defaults to a generic avatar when the user has not uploaded one.
</ResponseField>

<ResponseField name="[].createdAt" type="string" required>
  ISO 8601 timestamp of when the user account was created.
</ResponseField>

<ResponseField name="[].updatedAt" type="string" required>
  ISO 8601 timestamp of the last update to the user document.
</ResponseField>

<Note>
  The frontend uses this endpoint to populate the chat sidebar on load. Each item in the returned array represents a contact the authenticated user can open a conversation with.
</Note>

## Example request

```bash
curl -X GET https://your-api.com/api/message/users \
  -H "Authorization: Bearer your-jwt-token"
```

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | No valid Bearer token was provided in the `Authorization` header. |
| `500 Internal Server Error` | An unexpected error occurred on the server. |
